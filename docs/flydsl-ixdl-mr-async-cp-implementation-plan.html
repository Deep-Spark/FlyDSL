与旧稿差异（已按 CUTLASS/ixcc 校正）：

  • SME load 为 4 参数（sOffset, gBase(v4i32), gOffset, kop），不是 5 参数的 slbVaddr/glbVaddr/slbSoffset/glbSoffset 形态（后者属于 mllsa_async_*，非本 CopyAtom）。
  • gmem_ptr 在描述符内为 完整指针，非 addr_lo/hi 拆分；ivcore11 上 placeholder 通常 unused（ix30 才置 -1）。
  • xf 变种的 Dst swizzle layout 见 CUTLASS copy_traits_ix11_sme.hpp；原型 b32 row 可 Dst = Src。
  • Global 侧采用 SmeGmemFatPtr（gmem_ptr + stride_byte + byte_offset），在 FlyToIXDL 与 emitAtomCallSSA 间传递，不再在 CopyAtom 内临时拼描述符。

TableGen 三参数 (valBits, major, swizzleMod) 已确认；其余工程细节见 §6。

// soffset 以 element index 存储，需按 elemBits 转换为 byte offset
Value computeSoffset(int64_t elemBits) {
  if (elemBits == 8)  return soffsetRaw;
  if (elemBits % 8 == 0) return soffsetRaw * (elemBits / 8);
  return (soffsetRaw * elemBits) / 8;
}

// 方向判定（SSA 路径）
if (src is #fly_rocdl.buffer_desc) {
  // buffer → register (load)
  BufferFatPtr bp(srcPtrTy, src);
  ROCDL::RawPtrBufferLoadOp(copyTy, bp.bufferRsrc(), bp.swizzleByteOffset(), soffset, ...);
  return loaded;
}
if (dst is #fly_rocdl.buffer_desc) {
  // register → buffer (store)
  BufferFatPtr bp(dstPtrTy, dst);
  ROCDL::RawPtrBufferStoreOp(stored, bp.bufferRsrc(), bp.swizzleByteOffset(), soffset, ...);
  return stored;
}

3.3 Shared 侧地址

CUTLASS 将 smem 指针 cast 为 unsigned 传入 builtin 第一个参数（copy_ix11_sme.hpp）。Iluvatar 上可用 AS3 指针转 uint32（cute/arch/util.hpp 中 __ILUVATAR__ 分支）。Predication：!pred 时 smem_ptr = 0xffffff。

3.4 SmeGmemFatPtr（已确认：Iluvatar global fat pointer）

工程模式参照 BufferFatPtr（§2.5）；语义字段对齐 CUTLASS SmeDescriptor（§3.2）。已确认采用此方案，用于 !fly.memref<..., global> / #fly_ixdl.sme_gmem 在 FlyToIXDL 与 MRAsyncCp 之间的 lowering。

3.4.1 设计目标

  • 在 Fly 层区分「普通 global ptr」与「SME 异步拷贝用 global ptr」，避免 emit 时再从 bare !llvm.ptr<1> 反推 stride。
  • stride_byte 在 make_ptr / MemRef 降维时从 layout leading stride 写入 fat struct，与 CUTLASS Copy_Traits(sme_desc_(stride_byte)) 一致。
  • fly.add_offset 累加 字节偏移 到 fat ptr 字段 byte_offset；发射时合并进 gmem_ptr（smeDescriptorVec 用 bumped ptr），IXDL gOffset 恒为 0——避免硬件 gOffset 范围不足（对齐 CUTLASS operator+ + goffset=0，§6 已确认）。

3.4.2 Fly 地址空间（类比 buffer_desc）

2.3 变种映射（FlyDSL (valBits, major, swizzleMod) ↔ CUTLASS 参照 ↔ IXDL）

FlyDSL 契约：TableGen / Python 使用三参数；emitAtomCallSSA 按三元组选 IXDL::CpAsync_*，getThrBitLayoutDst 选 Layout_SME_*_{COLXFB*|ROWXFB*|INTER}_Atom（swizzleMod=1 时用 *_ROWXFB8_* 的 Swizzle_Mod 公式）。CUTLASS SMESwizzle 仅作对照（其 Col 在 half 路径仍绑 colxfb8，与下表 b16 col → colxfb16 不同）。

已确认（2026-05）

不采用 CUTLASS 四态枚举 SMESwizzle 作为 TableGen 主参数；用 (valBits, major, swizzleMod) 覆盖 ixcc 已暴露的 SME 变种，其中 swizzleMod 显式区分 XOR 与 Swizzle_Mod（Row8b 必须为 8,0,1）。原型实现仍可先只链 32,0,0。

  return FxLayout(FxC(64), FxC(1));  // 1 warp 协作
}

// ---- getThrBitLayoutSrc ----
Attribute CopyOpMRAsyncCpType::getThrBitLayoutSrc() const {
  return FxLayout(FxShape(FxC(64), FxC(128)), FxStride(FxC(1), FxC(1)));
}

// ---- getThrBitLayoutDst ----
Attribute CopyOpMRAsyncCpType::getThrBitLayoutDst() const {
  // 原型 (32,0,0)：Dst = Src；其余按 §3.5.3 选 COLXFB8/ROWXFB16/ROWXFB8/INTER Atom
  // swizzleMod==1 → Swizzle_Mod layout（CUTLASS ROWXFB8_*_Atom_Bits）
  return getThrBitLayoutSrc();
}

// ---- emitAtomCallSSA ----（src 为 SmeGmemFatPtr struct，见 §3.4）
FailureOr<Value> CopyOpMRAsyncCpType::emitAtomCallSSA(...) const {
  SmeGmemFatPtr srcFat(..., adaptor.getSrc());
  Value gBase = srcFat.smeDescriptorVec(rewriter, loc);  // vector<4xi32>
  Value gOffset = zero;  // tile 字节偏移已并入 gBase 内 gmem_ptr（§6）
  Value sOffset = castSmemPtrToI32(adaptor.getDst());
  // kop = CacheAll (0); switch (valBits, major, swizzleMod) → IXDL::CpAsync_*Op（§3.5.3）
  return Value();
}
