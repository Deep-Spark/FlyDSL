0. TL;DR & 整体分层架构

核心思想 — Type-erased Wrapper + Interface Dispatch: Fly Dialect 不直接编码任何硬件指令，它只持有一个 !fly.mma_atom<InnerMmaOp> 包装类型。InnerMmaOp 是后端 Dialect（如 fly_rocdl、未来的 fly_ixdl）定义的 纯类型，实现了 Fly_MmaOpTypeInterface，通过 emitAtomCall / emitAtomCallSSA 把自己 lower 成具体硬件 intrinsic（ROCDL MFMA、ROCDL WMMA、未来的 IXDL 张量核心 op…）。

因此新增一个硬件指令 = 在后端 Dialect 加一个 TypeDef + 实现 emitAtomCallSSA，Fly Dialect 与所有上层 Python DSL 不需要任何修改。

四个不变量记住：
  1. 所有 MMA 指令 lower 都通过 !fly.mma_atom 包装层进行；
  2. 具体指令类型（如 !fly_rocdl.cdna3.mfma<16x16x16,(f16,f16)->f32>）携带 shape + dtype，不携带 SSA value；
  3. fly.make_mma_atom Op 没有任何 operand，整个原子由 result type 编码；
  4. 有状态指令（如 CDNA4 MFMA-Scale 的 scale_a/b）通过 StatefulOpTypeInterface 维护 LLVM struct。

1. FlyDSL 中 MMA Atom 的整体设计

1.1 核心抽象（4 个层级）

FlyDSL 借鉴了 NVIDIA CuTe 的 atom 模型，自上而下分四层：

层级类型 / Op表达内容例子
① 指令实现 MmaOpTypeInterface 的具体类型一条硬件 MMA 指令的 MNK / dtype / 线程‑值布局 / 如何 lower!fly_rocdl.cdna3.mfma<16x16x16,(f16,f16)->f32>
② 原子!fly.mma_atom<InnerMmaOp>可调用的最小 MMA 单元，对外屏蔽硬件差异!fly.mma_atom<!fly_rocdl.cdna3.mfma<...>>
③ 平铺!fly.tiled_mma<atom, layout, perm>一个 block / warp 内多个 atom 的平铺方式!fly.tiled_mma<atom, (2,2,1), (M,N,K)>
④ 调用fly.mma_atom_call / fly.mma_atom_call_ssa / fly.gemm对 atom（或 tiled_mma）发起一次或多次 MMAfly.mma_atom_call(atom, %d,%a,%b,%c)

另有一个 通用回退 !fly.universal_fma<f32>（实现 MmaOpTypeInterface），lower 为 llvm.fmul + llvm.fadd，便于 CPU/无 MMA 硬件运行。

方法含义用途
getThrLayout参与单条指令的线程子布局（如 wave64 MFMA 是 64×1，wave32 WMMA 是 32×1）用于 tiled 拷贝/MMA 的线程映射
getShapeMNK静态 (M,N,K) 元组层布局推导
getValTypeA/B/C/D每个寄存器片段的标量元素类型fragment memref / SSA vector 类型推导
getThrValLayoutA/B/C(thread, value) → 在参考 tile 中的逻辑坐标tiled_mma.partition 决定每条线程持有的 tile 子集
emitAtomCallmemref 形式：从 ptr load A/B/C，做 MMA，store 回 D未启用 SSA 提升时的默认 lower 路径
emitAtomCallSSASSA 形式：A/B/C 已是 vector/标量，返回结果 vector启用 fly-convert-atom-call-to-ssa-form 后的快速路径

  3. fx.mma_atom_call(atom, d,a,b,c)
    → Op：fly.mma_atom_call %atom, %d, %a, %b, %c
  4. SSA 提升 pass 后
    → fly.mma_atom_call_ssa
  5. convert-fly-to-rocdl
    → MmaOpCDNA3_MFMAType::emitAtomCallSSA
  6. 最终
    → rocdl.mfma_f32_16x16x16f16 + llvm.store
