1. 整体架构:CopyAtom 四层模型与数据流

核心思想 — Type-erased Wrapper + Interface Dispatch: Fly Dialect 本身不编码任何硬件 load/store 指令.它只持有一个 !fly.copy_atom<InnerCopyOp, valBits> 包装类型.InnerCopyOp 是 Fly 通用类型(如 !fly.universal_copy<32>)或后端 Dialect(fly_rocdl)定义的纯 MLIR 类型,实现了 Fly_CopyOpTypeInterface,通过 emitAtomCall / emitAtomCallSSA 把自己 lower 成具体 IR(LLVM load/store、ROCDL buffer_load 等).

因此新增一种硬件拷贝指令 = 在后端 Dialect 加一个 TypeDef + 实现 CopyOpTypeInterface,Fly Dialect 与上层 Python DSL 通常无需修改.

2. Interface 体系:谁定义什么契约

Interface 是 FlyDSL Atom 设计的核心.TableGen 定义在 include/flydsl/Dialect/Fly/IR/FlyInterfaces.td,C++ 实现分散在各 CopyOp 类型对应的 *.cpp 中.

2.1 继承关系总览

  │     └── rebuildStaticValue()         ← fly-rewrite-func-signature Pass 用

每个方法的作用

方法返回/行为谁消费

所有 lower 逻辑都在 CopyOpTypeInterface::emitAtomCall* 中.CopyAtomType 只做转发,不决定最终生成 LLVM 还是 ROCDL 指令.

2.2 Fly_StatefulOpTypeInterface — 何时需要

硬件 buffer 指令常带 per-instruction 状态(如 CDNA3 soffset).Stateful CopyOp:

  • isStatic() == false(不能从函数签名移除后自动重建)
  • fly.make_copy_atom lower 为 getDefaultState() 返回的 LLVM struct,而非 llvm.undef
  • Python 通过 copy_atom.set_value("soffset", val) → fly.atom.set_value → setAtomState

状态字段枚举:include/flydsl/Dialect/FlyROCDL/IR/Atom.td 中的 AtomStateField(soffset、imm_offset、scale_a/b 等).

Op 定义

def Fly_MakeCopyAtomOp : Fly_Op<"make_copy_atom", [Pure]> {
  let arguments = (ins I32Attr:$valBits);
  let results = (outs Fly_CopyAtom:$result);
}

make_copy_atom 做两件事:① 构造 CopyAtomType.get(copyOp, valBits);② 插入 Pure op 产生 SSA 值.整个 atom 的语义(用哪种 CopyOp、元素位宽)全部编码在 result type 中.

Python DSL
copy_atom = fx.make_copy_atom(
    fx.UniversalCopy32b(), fx.Float32)
Tracing 生成的 Fly IR
%atom = fly.make_copy_atom {valBits = 32}
  : !fly.copy_atom<
      !fly.universal_copy<32>, 32>

Lowering(MakeCopyAtomOpLowering, lib/Conversion/FlyToROCDL/FlyToROCDL.cpp):stateful CopyOp → getDefaultState() 生成 LLVM struct;non-stateful → llvm.undef 空结构体占位.

fly.copy_atom_call vs fly.copy

fly.copy_atom_call 的 operand 类型是 Fly_CopyAtom(包装类型);fly.copy 的 operand 是 AnyType,因为 stateful lower 后 atom 已是 LLVM struct value.GEMM 等 kernel 中 fx.copy(copy_atom.set_value(...), src, dst) 走 fly.copy 路径.

4. C++ 实现:转发、Recast 与 emit

职责:纯转发 + layoutRecast

// TV layout: bit → value
Attribute CopyAtomType::getThrValLayoutSrc() {
  auto copyOp = cast<CopyOpTypeInterface>(getCopyOp());
  return layoutRecast(builder, copyOp.getThrBitLayoutSrc(), 1, getValBits());
}

// emit: 完全委托给内嵌 CopyOp
LogicalResult CopyAtomType::emitAtomCall(...) const {
  return cast<CopyOpTypeInterface>(getCopyOp())
      .emitAtomCall(builder, loc, copyAtomTy, srcMemTy, dstMemTy, atomVal, src, dst);
}

7. Pass Pipeline 与调用链

7.1 编译阶段一览

阶段Pass / 组件对 CopyAtom 做什么
Tracing @traced_op + InsertionPoint 插入 make_copy_atom、copy_atom_call 等 Fly ops
Layout fly-layout-lowering 展开 tiled copy 为 atom call 循环
SSA 提升 fly-convert-atom-call-to-ssa-form Register MemRef eligible 时:copy_atom_call → copy_atom_call_ssa
Vector SSA fly-promote-regmem-to-vectorssa 按 TV layout 将 register 标量提升为 vector SSA
Backend convert-fly-to-rocdl CopyAtomCall*Lowering → emitAtomCall* → LLVM/ROCDL
P fly-convert-atom-call-to-ssa-form
lib/Dialect/Fly/Transforms/ConvertAtomCallToSSAForm.cpp

当 src 或 dst 是 Register MemRef 且 layout 满足 isEligibleToPromote 条件时,Pass 将 MemRef 形式的 copy_atom_call 转为 SSA 形式:

  • eligible src:先 fly.ptr_load 取出 SSA 值
  • eligible dst:先 load old value,执行 copy_atom_call_ssa,再 fly.ptr_store 写回
  • 非 eligible 端:保持 MemRef 指针形式传给 SSA op
Before (MemRef 形式)
fly.copy_atom_call %atom, %src, %dst
  : !fly.copy_atom<...>,
    !fly.memref<f32, global, ...>,
    !fly.memref<f32, register, ...>
After SSA Pass
%src_ssa = fly.ptr_load %src : f32
%old = fly.ptr_load %dst_iter : f32
%new = fly.copy_atom_call_ssa
  %atom, %src_ssa, %old : f32
fly.ptr_store %new, %dst_iter

Promotion 条件(isEligibleToPromote):Register 地址空间,且 layout coalesce 后为 leaf,shape 或 stride 为 1.

L CopyAtomCallLowering / CopyAtomCallSSALowering
lib/Conversion/FlyToROCDL/FlyToROCDL.cpp:523–610

Lowering 不直接生成硬件指令,而是委托给 CopyAtomType::emitAtomCall*,再转发到内嵌 CopyOp 的 Interface 实现.

CopyAtomCallLowering::matchAndRewrite(CopyAtomCall op, ...) {
  copyAtom.emitAtomCall(rewriter, loc, ..., copyAtomVal, src, dst);
}

CopyAtomCallSSALowering::matchAndRewrite(CopyAtomCallSSA op, ...) {
  copyAtom.emitAtomCallSSA(rewriter, loc, resultTy, ..., src, dst);
}

CopyAtomCallLowering 校验 src/dst 均为 MemRef 且元素类型一致,有 pred 时走 predicated emitAtomCall.CopyAtomCallSSALowering 有返回值则 replaceOp,否则 eraseOp.

10.3 总结

要记住的 5 件事说明
1. 四层模型 CopyOp(指令语义) → CopyAtom(包装+valBits) → TiledCopy(平铺) → copy_atom_call(执行)
2. Interface 是扩展点 所有 lower 逻辑在 CopyOpTypeInterface::emitAtomCall*;CopyAtomType 只转发
3. bit vs value layout CopyOp 提供 bit-level TV;CopyAtom 用 valBits recast 为 value-level
4. 两条执行路径 MemRef:copy_atom_call + emitAtomCall;SSA:copy_atom_call_ssa + emitAtomCallSSA
5. Python 三层 MLIR 绑定 → primitive(@traced_op) → typing(CopyAtom/TiledCopy 增强)

一句话:CopyAtom 是 Fly 方言对「拷贝指令」的类型擦除包装;具体怎么 load/store 完全由内嵌 CopyOp 类型的 Interface 实现决定;Python 侧通过 make_copy_atom 组装类型、通过 copy_atom_call / copy 插入 IR,再经 SSA Pass 与 FlyToROCDL 落到 LLVM/ROCDL.
